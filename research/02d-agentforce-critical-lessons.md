# 补充：Agentforce 批判性复盘 —— 对我们最重要的战略教训

> 说明：本文件是"02 国外竞品"调研子代理产出，回传父代理失败，单独落盘。基于 Salesforce 自研基准 CRMArena-Pro + 大量独立/媒体来源（2025–2026）。所有正面客户指标均为厂商自报、未经审计。

## 核心教训（直接影响我们的架构与商业模式）

### 1. FSE 上门模式被市场领导者验证 ✅
- Salesforce 在 **Dreamforce 2025** 增设 **"forward-deployed engineers"（前置部署工程师）**去救火停滞的客户部署。
- SF 工程博客确认：真实部署原本要 **6 个月**，只有靠**重度上门支持**才压缩到 **3 周**；瓶颈是"不成熟的平台架构 + 复杂的企业数据环境"。
- **结论：你的 FSE 上门建 skill / 接系统的设想，与市场头部的收敛方向一致。**企业 agent 落地本质是"重服务"的，不是纯自助 SaaS。这既是成本项，也是护城河。

### 2. 自主 agent 被高估：设计成"窄任务 + 人在环 + 确定性兜底"
- **CRMArena-Pro（Salesforce 自研基准，2025-06）：**单轮任务成功率约 **58%**，多轮骤降到约 **35%**（即多轮任务失败约 65%）；且 agent **几乎零保密意识**，加提示词纠正保密反而拖累任务表现。
- Salesforce 自己悄悄**退回确定性/规则化控制**：推出 **Agent Script**（确定性对话层，2026-02 GA），因为 CTO 承认指令**超过 8 条**模型就会丢指令、出现"drift（漂移）"。
- **结论：**关键业务路径别赌纯 LLM 自主编排。用**确定性工作流/状态机**做骨架，LLM 只在节点内做理解与生成。我们那个"通用流程基座 + 各职位 skill"的设想方向对——但 skill 要尽量收敛成**可验证的窄工具**，而非开放式自主。

### 3. LLM 不负责安全，权限必须在平台层强制
- Salesforce 工程团队**官方确认**：语言模型只是"貌似合理地理解语言"，**不执行规则**；权限执行必须在平台层，靠 Agent User 的 profile/权限集/共享规则。
- Agent 继承的是**它被指派的那个用户身份**的 CRUD/字段级/共享权限，**不是**正在对话的终端用户的权限——若 Agent User 被授过宽权限，一次注入就能触达它能看到的一切（blast radius）。
- **ForcedLeak（Noma Security，2025-09 披露，CVSS 9.4 严重）：**针对开启 Web-to-Lead 的组织的间接提示注入链——线索描述字段藏指令，且 SF 的 CSP 白名单里有个**过期域名**（研究者花约 5 美元买下）用于外泄 CRM 数据，全程无需人工"发送"。已修复。
- **结论：**印证第 5 路授权调研——权限必须在**平台层用 OpenFGA/ReBAC 强制**，且 **agent 权限 ≤ 当前对话用户权限**（不能用一个胖服务账号）。同时把所有外部内容（文件/网页/字段）当**不可信输入**做提示注入隔离。

### 4. "拦截率 ≠ 解决率"，指标注水是头号信任障碍
- 生态普遍用 deflection（拦截/转移率）而非 resolution（真正解决率）作为 ROI 变量；审计发现 15–25% 被"拦截"的工单其实答案错误/不完整。
- Salesforce 自家 2026 State of Service：**71% 的支持负责人把"注水的自动化指标"列为信任 AI 厂商的头号障碍**。
- **结论：**我们对商户要用**诚实、可核验的指标**（真实解决率、人工接管率、错误率），并内建审计与回溯。这本身可作为差异化卖点。

### 5. 商业化现实：定价、成本、市场周期
- **定价反复：**$2/对话（2024-09）→ 因账单不可预测被 CFO 抵制 → Flex Credits（约 $0.10/action，2025-05）→ 又退回 **AELA 按人头约 $125/user/月**（2025-10）。
- **隐藏成本：**Data Cloud（grounding 走它的消耗计费）是事实上必需的隐藏大项；$125/user 的标价叠加后估算可达 **$960–1,870/user/月**（含 Data Cloud、Flex Credits、顾问，方向性估算）。
- **签约 ≠ 付费 ≠ 上生产：**约一半签约未付费；Business Insider 报道约 1.25 万客户里付费不到一半、实际用量偏低；Salesforce 把销售提成从"签约"改绑"实际用量"（默认承认签约虚高）。
- **市场周期（Gartner/Forrester）：****Gartner 预测 2027 年底前 >40% 的 agentic AI 项目将被取消**（成本、价值不清、风控弱），并警告"agent washing（智能体洗白）"；Forrester 称实际启用平台内 agentic 能力的组织不到 15%。
- **build vs buy：**据报道约 **100 万次对话/年**是分水岭——低于此买现成的更划算（速度/集成），高于此自建更划算（单 token 成本）。

## 竞争位对照（简）
- **vs Microsoft Copilot Studio：**微软优势是 M365/Azure 装机量 + agent 跑在客户自有租户内（数据驻留）；Agentforce 在客户已用 Salesforce/CRM 时占优。截至 2026，两者都不提供自托管运行时。
- **vs ServiceNow（Now Assist）：**IT/员工服务与流程自动化最强，基于既有身份/权限/审计；其 L1 服务台号称约 90% 自主解决 IT 请求。
- **vs 超大厂（Google Vertex/Gemini Enterprise、Amazon Bedrock AgentCore）：**builder 品类；Vertex 约 $12/1000 次查询。
- **护城河批评：**Agentforce 建在它不拥有的 OpenAI/Anthropic 模型之上；MCP/互操作 + 退回按人头定价，削弱了"按结果计费的数字劳动力"叙事。

## 一句话总结
市场领导者的真实经验告诉我们三件事：**(a) 落地靠重服务/FSE 上门**；**(b) 别赌自主，要窄任务 + 确定性兜底 + 人在环**；**(c) 安全与权限必须在平台层强制、指标必须诚实**。这三点应成为我们产品的立身之本，而非事后补丁。
