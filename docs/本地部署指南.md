# 本地部署指南 —— WSL 后端 + WSL mock + Windows 桌面

> 更新：2026-07-10 ｜ 对应 M5 实机验证的拓扑 ｜ 全本地/mock，零外部依赖（除 LLM 网关）

一句话拓扑：**WSL 里分开跑两个单元**（OpenFGA 授权库 docker 容器 + agentd Go 进程），**Windows 侧编译打包桌面 exe**，桌面连 WSL 里的 agentd（`localhost:8765`，走 WSL2 转发）。

```
Windows 桌面 exe ──HTTP/SSE──> localhost:8765 ──(WSL2 转发/::1)──> WSL agentd(0.0.0.0:8765)
                                                                      │
                                                          ┌───────────┴───────────┐
                                                   OpenFGA(docker :18080)   进程内内存 SQLite(ERP/CRM/skill)
```

> ERP/CRM/skill 是**嵌在 agentd 进程里的内存 SQLite**（启动时用内嵌 schema+seed 播种），不是独立服务 —— 这是设计选择。真正“分开部署”的 mock 是 **OpenFGA**（独立容器）。

---

## 0. 前置（本机已满足，换机时核对）

- **WSL**：Ubuntu 24.04，装 docker（daemon 在跑）、Go（本机在 `/home/alvin/.local/bin`，`GOROOT=/home/alvin/.local/opt/go1.26.x`）
- **Windows**：Node ≥ 22 + npm
- **网络**：中国网 —— Go 用 `goproxy.cn`、npm/Electron 用 `npmmirror`（下方命令已内置）
- **LLM 网关**：OpenAI 兼容地址 + api_key（本机用 `https://www.linx2.ai` / `gpt-5.5`）

## 1. 配置 `backend/dev.env`（一次性，gitignored 不入库）

```bash
# backend/dev.env —— 用 `set -a; . dev.env; set +a` 载入；Go 经 os.Getenv 读取
export LLM_BASE_URL=https://www.linx2.ai
export LLM_MODEL=gpt-5.5
export LLM_API_KEY=<你的 key>
export OPENFGA_API_URL=http://localhost:18080
export TENANT=mock-corp-001
export ADDR=0.0.0.0:8765                 # 绑 0.0.0.0，Windows 才连得到（见 §5 网络说明）
export MOCK_ORG_FILE=/mnt/d/merchantAgent/backend/testdata/mock-org.yaml   # 绝对路径
```

> `.gitignore` 用 `dev.env` + `*.env` 双规则挡住，已验证密钥不入 git。

---

## 2. WSL —— 起 OpenFGA（mock 授权库）

```bash
cd /mnt/d/merchantAgent/backend
docker compose up -d                     # OpenFGA -> host :18080（8080 被本机占用，故用 18080）
# 等就绪：
curl -fsS http://localhost:18080/healthz # {"status":"SERVING"}
```
> 内存 datastore，重启即清空；agentd 每次启动都会重新播种，无需手动初始化。healthcheck 可能显示 unhealthy（gRPC 探针），不影响 HTTP API。

---

## 3. WSL —— 编译 + 运行 agentd

```bash
export PATH="/home/alvin/.local/bin:$PATH"
export GOPROXY=https://goproxy.cn,direct GOSUMDB=off      # 中国网必需
cd /mnt/d/merchantAgent/backend
go build -o ~/merchantagent/agentd ./cmd/agentd           # 编到 home，别在 /mnt 跑
# 载入配置并后台常驻（detached，退终端不掉）：
set -a; . /mnt/d/merchantAgent/backend/dev.env; set +a
setsid ~/merchantagent/agentd </dev/null >~/merchantagent/agentd.log 2>&1 &
sleep 2; cat ~/merchantagent/agentd.log                   # 见 "listening on 0.0.0.0:8765"
```

可选自测（连真 OpenFGA 跑全测）：`go test ./...`（11 包应全绿）。

## 4. Windows —— 编译 + 打包桌面 exe

在 Windows 侧（git-bash / PowerShell 均可，下面是 bash）：

```bash
cd /d/merchantAgent/desktop
export ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
export ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
export CSC_IDENTITY_AUTO_DISCOVERY=false                  # 本地不签名

npm install --registry=https://registry.npmmirror.com     # 中国网必需；勿中途 kill（见坑2）
npm run build                                             # electron-vite 三端打包 -> out/
npm run dist:win                                          # electron-builder 出 NSIS 安装包
```

产物在 `desktop/dist/`：
- `merchantAgent Setup 0.1.0.exe` —— NSIS 安装包（≈82MB）
- `win-unpacked/merchantAgent.exe` —— 免安装版

### 打包已知坑（本机已踩平，换机可能复现）

1. **winCodeSign 符号链接权限失败**（`Cannot create symbolic link ... darwin/...dylib`）：Win10 无 Developer Mode/管理员时无法建符号链接，而那是 macOS 文件、Windows 打包用不上。预解压排除 darwin 即可：
   ```bash
   C="$LOCALAPPDATA/electron-builder/Cache/winCodeSign"   # 或 ~/AppData/Local/...
   7za x "$C"/*.7z -o"$C/winCodeSign-2.6.0" -xr!darwin -y  # 用 node_modules/7zip-bin 里的 7za
   # 之后再跑 dist:win，electron-builder 会直接用这份缓存、跳过它自己会失败的解压
   ```
2. **npm install 被中断**会留下残缺 node_modules（`7zip-bin`/`app-builder-bin` 空目录、`.bin` 缺 shim）→ 重新完整 `npm install`（走 mirror）补齐。

## 5. 启动 + 验证

装好/解压后双击 `merchantAgent.exe`（**先保证 §2、§3 的 OpenFGA + agentd 在跑**）。切换顶部身份问 “SO-1001 进度/利润/齐套”，看**同问不同权**。

命令行冒烟（可选）：
```bash
curl -X POST http://localhost:8765/login -d '{"userId":"u_sales1"}'
curl -N -X POST http://localhost:8765/chat -H 'content-type: application/json' \
  -d '{"sessionId":"s","userId":"u_smgr","question":"SO-1001 的利润是多少"}'
```

**网络说明（重要）**：桌面连的是 `localhost:8765` 而非 `127.0.0.1:8765`。本机 WSL2 的 localhost 转发是**纯 IPv6**（`.wslconfig` 的 `mirrored` 在 Win10 被忽略），`127.0.0.1` 字面量连不上、`localhost`/`::1` 可达；代码已用 `localhost`（Node fetch autoSelectFamily 自动选通的族），勿改回。mock 用户：`u_sales1`(销售) / `u_smgr`(销售经理,看成本) / `u_boss`(总经理) / `u_fin`(会计,看成本) / `u_plan`(计划员,无 skill 被拒)。

---

## 6. 重启（WSL 重启或关机后）

```bash
# WSL 里，按顺序：
cd /mnt/d/merchantAgent/backend && docker compose up -d
set -a; . /mnt/d/merchantAgent/backend/dev.env; set +a
setsid ~/merchantagent/agentd </dev/null >~/merchantagent/agentd.log 2>&1 &
# 然后 Windows 双击 merchantAgent.exe
```
桌面 exe 已装好就不必重打；只有改了 `desktop/` 源码才需重跑 §4。
