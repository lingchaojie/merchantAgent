## 5. 桌面壳（Electron + Vite + React + TS）— 已实现

对标 Cursor/Codex 架构：**Electron UI 壳 + 编译型后端(Go agentd)干重活**。栈：electron-vite + React 18 + TypeScript(严格) + Vitest。

**结构与安全**：
- `src/shared/contract.ts`：★ IPC 契约唯一真源（`AgentAPI/Answer/Principal/Channels`），main/preload/renderer 三端共享，形状漂移即编译报错。
- `src/main/`：`index.ts`(安全壳：contextIsolation + sandbox + nodeIntegration:false + CSP 仅连 loopback + 拦截外链)、`ipc.ts`(唯一特权桥)、`fsguard.ts`(路径沙箱：防 `../` 逃逸、覆盖写需确认)、`agentd.ts`(连/spawn Go 后端)。
- `src/preload/index.ts`：contextBridge 暴露类型化 `window.agent`（渲染层零 Node）。
- `src/renderer/`：React UI。

**UI（Linear 风格，已实现）**：
- 设计系统 `theme.css`：深色近黑 `#0b0c0e`、靛紫强调 `#5e6ad2`、发丝边框、Inter、圆角 tokens。
- 组件：`Sidebar`(工作区头 + 会话列表 + 底部身份 + mock 标识)、`TopBar`(标题 + 身份 chip + Cmd+K)、`CommandPalette`(Cmd+K 键盘可导航：切身份/新建会话)、`ChatView`(消息行 + 头像 + **工具徽章** + **权限拦截锁态** + pending 打字动画)、`ResultCard`(按工具渲染结构化卡片：进度/财务/齐套欠料)、`EmptyState`(示例 prompt 芯片)、`Composer`(自增长 textarea，Enter 发送)。

**已验证**（typecheck + build + Playwright 程序化核验）：
- `npm run typecheck` 双端 0 错误；`npm run build` electron-vite 三端打包成功（41 模块）；`vitest` fsguard 5 用例绿。
- Playwright 程序化确认：Linear tokens 全部生效（布局 248px 侧栏、`#0b0c0e` 背景、`#5e6ad2` 强调、Inter）；点示例→出消息+结构化卡片；销售问利润→**权限拦截锁态**；切总经理问利润→**同一问题变为通过 + 财务卡片(利润绿色)**；Cmd+K 面板正常。
- Electron 在本 WSL 用 `--disable-gpu --no-sandbox`（WSL workaround）干净启动、main 跑到位、零渲染层报错。

**运行**：
